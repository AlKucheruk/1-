# Этап 8 — Загружаем платёжки в 1С

## Цель этапа

Перенести банковские платежи из `1С БП` в `1С УТ` так, чтобы:

- документы в УТ создавались в корректном виде (`Поступление`/`Списание`);
- сохранялась аналитика по контрагентам, договорам и статьям ДДС;
- исключалась повторная загрузка одних и тех же платежей.

## Состав этапа (файлы в этой папке)

| Файл | Назначение |
|------|------------|
| `Код1СУТ_08_Загрузка Платежек.bsl` | Модуль формы внешней обработки загрузки (вставляется в конфигураторе в обработку ниже). |
| `Код1СУТ_08_ЗагрузкаПлатежекИзБП.epf` | Скомпилированная внешняя обработка для запуска в УТ. |
| `03_modefide.py` | Подготовка выгрузки банка: нормализация, маппинг операций и статей ДДС (нужны рядом `mapping_operations_ut.xlsx`, `dds_mapping_ut_bp.xlsx` — см. скрипт). |
| `04_разбиение данных.py` | Разбиение подготовленного файла по периодам. |
| `ИНСТРУКЦИЯ_ФОРМА_Этап8_Загрузка_Платежей.md` | Соответствие имён реквизитов формы и модуля. |

Старые копии кода, архивы и вспомогательные каталоги из репозитория убраны; актуальная логика — в перечисленных файлах.

## Результат этапа

В УТ появляются платёжные документы с прослеживаемой связью с источником из БП (в т.ч. регистр связей по УИД, см. `1c_ut_knowledge.rules`, раздел 8).

## Усиления обработки (апрель 2026)

- Ускорен предпросмотр (кэши по ИНН, организациям, счетам, статьям, УИД).
- Дубли по УИД проверяются на предпросмотре и повторно перед записью; связь в регистре и документ — в одной транзакции.
- Нормализация договоров в духе этапа 3; обработка счетов-заглушек; финальные инварианты суммы/контрагента; FIFO по договору.

## Baseline и приёмка

### Базовые замеры (до/после)

По каждому тест-файлу: время предпросмотра и загрузки, итог `Создано/Пропущено/Ошибок`, строки `Готов/Дубль/Ошибка`.

Рекомендуемые объёмы: малый (~100–500 строк), средний (~3k–10k), большой (50k+ при возможности).

### Чеклист приемки

- Повторный запуск того же файла не создаёт новые документы (контур дублей).
- Для строк с одной заглушкой у разных клиентов — разные техсчета.
- В ссылочные поля не попадают «сырые» строки вместо ссылок.
- При ошибке инварианта откатывается текущая строка.
- Сумма документа = сумма ТЧ `РасшифровкаПлатежа`.
- FIFO не выходит за договор.
