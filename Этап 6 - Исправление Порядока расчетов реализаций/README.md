# Этап 6 — Порядок расчетов в реализациях

Папка — **полная копия этапа 4** (внешняя обработка, те же реквизиты и форма в конфигураторе). В модуль формы вставляйте код из **`Код1С_УТ_06_ПорядокРасчетов_Реализации_модульФормы.bsl`**: меняется только логика (установка `ПорядокРасчетов = ПоДоговорамКонтрагентов`, команда применения в коде: **`ИсправитьПорядокРасчетов`**; в форме обработки укажите то же имя действия). Перед реализацией при необходимости то же значение выставляется в связанном **`ЗаказКлиента`**, если у заказа есть реквизит `ПорядокРасчетов`. Прежний BSL с соглашениями и договорами переименован в **`Код1С_УТ_04_Исправление Реализаций_АРХИВ_логика_соглашение_договор.bsl`**.

Лог этапа 6: `D:\1с\КАА\Лог_06_ПорядокРасчетов_Реализации.txt`.

Перед прогоном: **`ПРОВЕРКА_ЭТАЛОНА_ПОРЯДОК_РАСЧЕТОВ.txt`**. Порядок пилота и контроля: **`ПИЛОТ_И_КОНТРОЛЬ.txt`**. На форме обработки можно добавить реквизит **`ТолькоПоРасчетнымДокументам`** (Булево): при значении «истина» в предпросмотр попадают только документы с `ПорядокРасчетов = ПоРасчетнымДокументам`; без реквизита действует широкий отбор «всё, что не целевое значение».

В **`ТаблицаДанных`** код заполняет **`Реализация`**, **`Заказ`**, **`ДатаРеализации`**, сумму, контрагента, **`ПорядокРасчетов`** (тип `ПеречислениеСсылка.ПорядокРасчетов`), при наличии колонки — строку **`ТекущийПорядокРасчетов`**. Пошагово по колонкам: **`Реквизиты_ТаблицаДанных_форма.txt`**. Колонки **`Соглашение`**, **`Договор`**, **`ДоговорКонтрагента`** не используются — удалите их в конфигураторе.

Синхронизация **`ЗаказКлиента`**: при ошибке проведения заказа (объект расчётов, остатки и т.д.) реализация всё равно обрабатывается; в комментарии строки — предупреждение по заказу. Полный отказ от записи заказа — реквизит формы **`НеСинхронизироватьЗаказКлиента`** (см. **`Реквизиты_ТаблицаДанных_форма.txt`**, §5).

# Код1С_04.bsl - аннотация (исходное описание копии этапа 4)

Ниже — описание **оригинальной** логики этапа 4; в рабочей обработке этапа 6 оно **не** используется.

Этот модуль внешней обработки предназначен для массовой корректировки документов `РеализацияТоваровУслуг`: подтягивает и устанавливает `Соглашение` и `Договор` в реализациях (и связанных заказах клиента), с предпросмотром строк и выборочной обработкой.

## Что делает обработка

- Формирует предпросмотр документов за период (с фильтром по контрагенту и опцией `ПЕРВЫЕ 100`).
- Для каждой строки показывает текущие значения `Соглашение/Договор`, контрагента, сумму и служебный комментарий.
- Если у реализации нет договора, пытается подобрать договор контрагента типа `СПокупателем`.
- Позволяет массово отметить/снять строки для обработки (`ВыбратьВсе` / `СнятьВсе`).
- По кнопке применения:
  - для отмеченных строк устанавливает выбранное на форме `Соглашение`;
  - устанавливает договор (текущий в документе или подобранный `ДоговорКонтрагента`);
  - обновляет и проводит `ЗаказКлиента` (если есть) и `Реализацию`;
  - проверяет, что сумма документа не изменилась после перезаписи.

## Логика безопасности

- Обработка каждой строки идет в отдельной транзакции.
- При ошибке выполняется `ОтменитьТранзакцию()`, в строку пишется текст ошибки.
- Успешные строки помечаются `OK`.
- Если документ уже заполнен нужным соглашением и договором, строка пропускается.

## Логирование

- Все этапы пишутся в файл:
  - `D:\1с\КАА\Лог_Обработка_Соглашения.txt`
- Лог включает старт/финиш этапов, параметры отбора, статусы `УСПЕХ/ОШИБКА/ПРОПУЩЕНО`.

## Ключевые процедуры

- `ЗагрузитьПредпросмотрНаСервере()` - загрузка и подготовка таблицы предпросмотра.
- `ПодтянутьСоглашенияНаСервере()` - массовое применение изменений в документах.
- `ЗаписатьЛог()` - запись строк в лог-файл.

## Связанные файлы в текущей структуре

- `Материалы из src/Добавление_Соглашений_В_Старые_Реализации.epf` - файл внешней обработки из папки `src`, соответствует логике этого этапа.
