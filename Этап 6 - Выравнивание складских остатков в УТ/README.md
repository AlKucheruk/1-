# Этап 6 - Выравнивание складских остатков в УТ

Цель этапа: найти моменты, когда товар уходил в минус по складу.

На этом шаге готовим плоскую выгрузку движений из УТ. Анализ (поиск минусов, расчет кумулятивного остатка) делаем отдельно в Python на следующем шаге.

## Что выгружаем

Плоская таблица без группировок в разрезе движений:

- `Период` (дата+время движения)
- `Склад`
- `КодНоменклатуры`
- `Номенклатура`
- `Регистратор` (документ)
- `Операция` (`Приход`/`Расход`)
- `Количество`

## Параметры периода

Обязательно ограничивать период параметрами:

- `ДатаНач`
- `ДатаКон`

Рекомендуемый режим запуска:

- сначала помесячно;
- если объем маленький, можно поквартально.

## Файлы этапа

- `Запрос_к_УТ_Этап_6_Поток_движений.txt` - основной запрос для СКД.
- `ИНСТРУКЦИЯ_СКД_Этап_6.md` - настройка СКД для выгрузки плоской таблицы.

## Важно

- Не включать группировки и итоги в СКД.
- Сортировка в выводе: `Склад, КодНоменклатуры, Период, Регистратор`.
- Формат выгрузки: `xlsx` (или `csv`), одна строка = одно движение.
