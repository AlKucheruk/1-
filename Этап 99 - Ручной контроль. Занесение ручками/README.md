# Этап 99 - Ручной контроль и занесение вручную

## Назначение этапа

Этот этап закрывает исключения, которые не удалось надежно обработать автоматикой на предыдущих шагах.

Здесь фиксируются платежи, требующие ручной проверки и ручного занесения в `1С УТ`.

## Что находится в этапе

- `Платёжки которые отконтралировать и занести вручную.xlsx` - список документов для ручной отработки.

## Когда сюда попадают записи

Как правило, сюда выносятся случаи:
- не найден или неоднозначен контрагент/договор/статья ДДС;
- недостаточно данных в назначении платежа;
- конфликт по аналитике или сомнительное сопоставление, которое нельзя автоматически подтвердить.

## Ожидаемый результат

После ручной обработки этих платежей:
- закрываются все "хвосты" автоматической загрузки;
- итоговые обороты и взаиморасчеты в УТ приводятся к целевому состоянию.
