# Код1С\_02\_ОбновлениеКонтрагентовПоКоду.bsl - аннотация

Этот модуль внешней обработки предназначен для нормализации карточек контрагентов в УТ по данным из Excel: загружает предварительную таблицу, находит контрагентов по идентификатору, показывает статус сопоставления и массово обновляет реквизиты контрагента/партнера.

## Что делает обработка

* Загружает Excel-файл в таблицу на форме (`Название\\\_ЕГР`, `ОГРНИП`, `ВидКонтрагента`, `Клиент`, `Поставщик`, `ИНН`, `Код`).
* На этапе загрузки пытается сразу найти ссылку на контрагента:

  * основной поиск: по `Партнеры.Код` через связь `Контрагенты.Партнер`;
  * fallback: по `Контрагенты.РегистрационныйНомер`.
* Заполняет комментарий строки статусом сопоставления:

  * `Найден при загрузке (по идентификатору)`;
  * `Не найден при загрузке`.
* Позволяет открыть карточку контрагента из строки таблицы (если есть колонка `Контрагент` и ссылка заполнена).
* По кнопке `Обновить в 1С` обрабатывает отмеченные строки и обновляет только отличающиеся значения.

## Какие поля обновляет

* В `Справочник.Контрагенты`:

  * `Наименование`, `НаименованиеПолное`;
  * `ИНН`;
  * `РегистрационныйНомер` (из колонки `ОГРНИП`);
  * `ЮрФизЛицо`;
  * `ЮридическоеФизическоеЛицо` (служебная синхронизация).
* В `Справочник.Партнеры` (если есть связанный партнер):

  * `Наименование`, `НаименованиеПолное` (рабочее отображение в режиме "Партнеры как контрагенты");
  * `Клиент`, `Поставщик`;
  * `ЮрФизЛицо` (через перечисление `КомпанияЧастноеЛицо`).

## Преобразование вида контрагента

* По тексту `ВидКонтрагента`:

  * `ип` / `индивидуаль` -> `ИндивидуальныйПредприниматель`;
  * `физ` -> `ФизЛицо`;
  * `нерезидент` -> `ЮрЛицоНеРезидент`;
  * `юр` -> `ЮрЛицо`.
* Если текст нераспознан, используется fallback по длине ИНН:

  * 12 -> `ИндивидуальныйПредприниматель`;
  * 10 -> `ЮрЛицо`.

## Логика безопасности и обработки

* Обрабатываются только строки с флагом `Обработать = Истина`.
* Для каждой строки:

  * ищется контрагент (из ссылки строки или повторным поиском);
  * если не найден - строка помечается как `Не найден по коду`;
  * если найден - обновляются только изменившиеся поля.
* Код идентификатора не перезаписывается в карточке, используется только для поиска.
* По итогам формируется статистика: обработано, обновлено, без изменений, не найдено, ошибок.

## Логирование

* Лог пишется в:

  * `D:\\\\1с\\\\КАА\\\\Лог\\\_Этап3\\\_ОбновлениеКонтрагентовПоКоду.txt`
* Логируются:

  * старт загрузки и обновления;
  * количество найденных/не найденных при загрузке;
  * изменения по каждой строке;
  * ошибки и итоговая статистика с временем выполнения.

## Ключевые процедуры и функции

* `ЗагрузитьПредварительныеДанные()` / `ЗагрузитьПредварительныеДанныеНаСервере()` - загрузка Excel и подготовка таблицы.
* `ОткрытьКарточкуКонтрагента()` - переход в карточку из выбранной строки.
* `ОбновитьВ1СНаСервере()` - массовое применение изменений.
* `ОбновитьКонтрагентаИПартнера()` - сравнение и запись реквизитов.
* `НайтиКонтрагентаПоИдентификатору()` - поиск по коду партнера с fallback по регистрационному номеру.
* `ЗаписатьЛог()` - запись в лог-файл.
