# Инструкция запуска обработки Этапа 3

Файл обработки: `Код1С_03_УТ_СозданиеИДоЗаполнениеДоговоров.bsl`.

## 1) Что создать в форме обработки

Реквизиты формы:
- `ПутьКФайлуExcel` - Строка(500)
- `ИнфоТаймер` - Строка(500)
- `ТаблицаПредварительныхДанных` - ТаблицаЗначений с колонками:
  - `Обработать` (Булево)
  - `НаименованиеКонтрагента` (Строка 250)
  - `ИНН` (Строка 12)
  - `КПП` (Строка 9)
  - `Код` (Строка 30)
  - `Контрагент` (СправочникСсылка.Контрагенты)
  - `Клиент` (Строка 3)
  - `Поставщик` (Строка 3)
  - `ПометкаУдалКонтрагента` (Строка 3)
  - `Комментарий` (Строка 1000)

Команды формы:
- `ЗагрузитьДанныеИзExcel`
- `ОбновитьДанныеВ1С`
- `ВыделитьВсе`
- `СнятьВсе`

## 2) Требования к Excel

На первом листе (Лист_1) должны быть колонки:
- `НаименованиеКонтрагента`
- `ИНН`
- `КПП`
- `Код`

## 3) Порядок запуска

1. Открыть обработку.
2. Нажать `Загрузить данные из Excel`.
3. Проверить таблицу предпросмотра:
   - найден ли `Контрагент`;
   - признаки `Клиент` / `Поставщик`;
   - комментарии по строкам.
4. Отметить строки для обработки (`Обработать = Истина`).
5. Нажать `Обновить данные в 1С`.
6. После завершения проверить:
   - `ИнфоТаймер` на форме;
   - лог `D:\1с\КАА\Лог_Этап3_СозданиеИДоЗаполнениеДоговоров.txt`.

## 4) Что делает обработка

- Ищет менеджера `Кучерук Алексей Александрович` один раз и использует для всех договоров.
- Для контрагентов:
  - если `Клиент = Да` - обрабатывает договоры типа `СПокупателем`;
  - если `Поставщик = Да` - обрабатывает договоры типа `СПоставщиком`.
- Если нужного договора нет - создает.
- Если есть - проверяет целевые поля и меняет только несоответствия.
- По каждому договору отдельная транзакция: при ошибке откат и переход к следующему.

## 5) Минимальный smoke-test

Сначала прогнать 3 строки:
- только `Клиент = Да`;
- только `Поставщик = Да`;
- строка, где уже все корректно.

Ожидаемо:
- в 1 и 2 случае договор создается или обновляется;
- в 3 случае договор не перезаписывается (в логе "без изменений").
