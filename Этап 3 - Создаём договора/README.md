# Этап 3 - Создаём договора (финальная версия)

## Возможности внешней обработки:

Массово создать/дозаполнить договоры контрагентов в `1С УТ`по заранее продопределённым правилам.

## Основные файлы этапа

- `УТ_Клиенты_ALLShorts.xlsx` - входной список контрагентов.
- `Код1С_03_СозданиеИДоЗаполнениеДоговоров.bsl` - рабочая внешняя обработка.
- `ИНСТРУКЦИЯ_ЗАПУСКА_ОБРАБОТКИ.md` - детальная инструкция по форме и запуску.

## Как работает обработка

- Из 1С через УправляемуюКонсольОтчетов получаем полный список клиентов нужный для загрузки в данную обработку. Название запроса (Запрос_к_УТ_Контрагенты_короткий вариант.osi) 
- Загружает Excel (`НаименованиеКонтрагента`, `ИНН`, `КПП`, `Код`) и находит контрагентов в УТ.
- Для каждой отмеченной строки определяет, обрабатывать как клиента и/или поставщика.
- Для `СПокупателем` и `СПоставщиком`:
  - если договора нет - создает;
  - если есть - обновляет только отличающиеся поля.
- Поддерживает обязательные и скрытые поля УТ (в т.ч. для детализации расчетов и закупки).
- По каждому договору выполняет отдельную транзакцию с откатом при ошибке.

## Пошаговая работа с обработкой

1. Открыть внешнюю обработку `Код1С_03_СозданиеИДоЗаполнениеДоговоров.bsl`.
2. Нажать `ЗагрузитьДанныеИзExcel` и выбрать входной файл.
3. Проверить предпросмотр: найденный `Контрагент`, признаки `Клиент/Поставщик`, комментарии.
4. Отметить строки к обработке (`Обработать = Истина`).
5. Нажать `ОбновитьДанныеВ1С`.
6. Проверить результат:
   - комментарии по строкам в таблице;
   - итог `ИнфоТаймер`;
   - лог-файл `D:\1с\КАА\Лог_Этап3_СозданиеИДоЗаполнениеДоговоров.txt`.

## Логирование и разбор ошибок

- Перед каждой новой сессией в лог добавляются 2 пустые строки (визуальный разделитель).
- После обработки формируется блок:
  - `=== Контрагенты с ошибками (для ручной доработки) ===`
- Формат строки для ручного поиска:
  - `Клиент <Наименование> | ИНН <ИНН> | обновление договора не состоялось. <Причина>`

Это позволяет быстро отфильтровать контрагентов, которые не прошли обновление (например, из-за блокировок или конкурирующей работы пользователей).

## Результат этапа

После выполнения этапа в УТ есть валидные договорные связки для клиентов/поставщиков, а все проблемные строки вынесены в отдельный блок лога для ручной корректировки.
