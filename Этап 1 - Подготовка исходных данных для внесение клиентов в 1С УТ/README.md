# Этап 1 - Подготовка исходных данных для внесения клиентов в 1С УТ

## Цель этапа

Сформировать корректный список клиентов для дозаполнения в `1С УТ` на основе сравнения данных `1С БП` и `1С УТ`.

## Что используется

* `Запрос_БП_Этап1_Контрагенты_и_банк_для_сравнения.txt` — выгрузка контрагентов и банковских реквизитов из **1С:БП 3.0** (консоль запросов / отчёт).
* `Запрос_УТ_Этап1_Контрагенты_и_банк_для_сравнения.txt` — выгрузка из **1С:УТ 11** (аналогично).
* `Сравнение клиентов в базах УТ и БП.py` — сравнение и дозаполнение в Excel (листы «Клиенты которых нет в УТ» и «UT_bank»; таблицы `tbl_UT`, `tbl_ACC` и т.д.). В начале файла задайте переменную **`FILE`** — путь к книге с выгрузками БП/УТ. Ранее в корне репозитория лежал дубликат `process_clients.py` (тот же код, другой путь к тестовому xlsx) — удалён, правьте только этот скрипт в папке этапа.
* Рабочие Excel-файлы подготовки списков (имена в проекте могут отличаться; см. `*.xlsx` в папке и в корне репозитория).
* Внешние обработки (скомпилированные, `.epf`):
  * `KAA_УТ_Этап_1_Пакетная_Загрузка_Клиентов.epf` — основная;
  * `Код1СУТ_01_Пакетная_Загрузка_Клиентов.epf` — копия из корня репозитория (тот же сценарий, другое имя при выгрузке из конфигуратора).

## Логика этапа

1. Выполняются запросы к БП и УТ.
2. Результаты сохраняются в один Excel-файл по разным листам.
3. Python-скрипт сравнивает клиентов (по ИНН), находит отсутствующих в УТ.
4. Для клиентов с пустыми банковскими реквизитами в УТ подтягиваются данные из БП.
5. Формируются итоговые листы в Excel для дальнейшей загрузки/обновления.
6. Для дальнейшей работы в 1С лист с клиентами для загрузки сохраняют в отдельной книге (один лист).

## Результат этапа

Подготовлен файл с проверенными клиентами и банковскими реквизитами для следующих этапов.

## Работа в 1С

1. Загрузка новых клиентов: внешняя обработка `KAA_УТ_Этап_1_Пакетная_Загрузка_Клиентов.epf` (или `Код1СУТ_01_...epf`).
2. Загрузить подготовленный Excel, отметить флажками строки к загрузке.

По итогу этапа из списка в Excel создаются новые клиенты и по цепочке — нужные договоры.
