# Этап 1 - Подготовка исходных данных для внесения клиентов в 1С УТ

## Цель этапа

Сформировать корректный список клиентов для дозаполнения в `1С УТ` на основе сравнения данных `1С БП` и `1С УТ`.

## Что используется

* `Запрос\\\\\\\_к\\\\\\\_БП.txt` - запрос для выгрузки клиентов и банковских реквизитов из БП.
* `Запрос\\\\\\\_к\\\\\\\_УТ.txt` - запрос для выгрузки клиентов из УТ.
* `Сравнение клиентов в базах УТ и БП.py` - Python-скрипт сравнения и дозаполнения.
* `ФайлДороботкиКлеинтов\\\\\\\_в\\\\\\\_УТ.xlsx` - рабочий файл с входными и выходными листами.
* `KAA\\\\\\\_УТ\\\\\\\_Этап\\\\\\\_1\\\\\\\_Пакетная\\\\\\\_Загрузка\\\\\\\_Клиентов` - внешняя обработка пакетного создания/дозаполнения клиентов в УТ.

## Логика этапа

1. Выполняются запросы к БП и УТ.
2. Результаты сохраняются в один Excel-файл по разным листам.
3. Python-скрипт сравнивает клиентов (по ИНН), находит отсутствующих в УТ.
4. Для клиентов с пустыми банковскими реквизитами в УТ подтягиваются данные из БП.
5. Формируются итоговые листы в Excel для дальнейшей загрузки/обновления.
6. Для дальнейшей работы в 1С нужно лист с клеинтами которых будем загружать сохранить в новой книге где будет 1 лист.

## Результат этапа

Подготовлен файл с проверенными клиентами и банковскими реквизитами, который используется в следующих шагах проекта.

## Работа на стороне 1С.

1. Для загрузки новых клиентов в 1С запускаем в программе 1С внешнюю обработку KAA\_УТ\_Этап\_1\_Пакетная\_Загрузка\_Клиентов.
2. Загружаем подготовленный файл Excel. Проверяем и проставляем чек-боксы тех клиенотов кого загружаем в базу 1С.
